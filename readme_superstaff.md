# คู่มือการใช้งานระบบ skinEPA เวอร์ชั่น 2.20 (Comprehensive User Manual)

ระบบ **skinEPA Tracker V 2.20** เป็นแพลตฟอร์มดิจิทัลที่พัฒนาขึ้นบนโครงสร้างเทคโนโลยี **LINE Front-end Framework (LIFF)** เชื่อมต่อฐานข้อมูลประสิทธิภาพสูงแบบ Real-time ร่วมกับ **Google Apps Script (GAS)** และ **Google Sheets** มีวัตถุประสงค์หลักเพื่อบันทึก ประเมิน และติดตามความก้าวหน้าทางการศึกษาของแพทย์ประจำบ้าน (Resident) และแพทย์ประจำบ้านต่อยอด (Fellow) ทั้งในส่วนของหัตถการหลัก (EPA), การเข้าร่วมกิจกรรมวิชาการ (Academic Attendance) และการประเมินผลรอบทิศทาง 360 องศา โดยเวอร์ชั่น 2.20 นี้ได้พัฒนาโครงสร้างความปลอดภัย ระบบคัดกรองข้อมูลโดยอาจารย์ที่ปรึกษา และระบบสื่อสารความเร็วสูงผ่าน LINE API อย่างสมบูรณ์

---

## 🔐 โครงสร้างการเข้าสู่ระบบและการยืนยันตัวตน (Authentication Flow)
ระบบใช้กลไกการยืนยันตัวตนแบบ **LINE UID Binding Cross-Reference** ควบคุมป๊อปอัปแจ้งเตือนทั้งหมดด้วยโปรแกรม **SweetAlert2 (Swal)** ซึ่งมีลอจิกการทำงานดังนี้:
1. **Auto-Login:** เมื่อเปิดแอปพลิเคชันผ่าน LINE ระบบจะดึงรหัสรหัสลับประจำตัว `line_uid` ของแอปพลิเคชัน LINE อัตโนมัติ หากรหัสนั้นเคยผูกไว้ในฐานข้อมูลชีต `staff` แล้ว ระบบจะอนุญาตให้ข้ามหน้าจอลงทะเบียนและเข้าสู่เมนูหลักตามสิทธิ์ (Role-Based Access Control) ทันทีภายใน 1 วินาที
2. **First-Time Login:** หากเป็นผู้ใช้งานใหม่หรือยังไม่เคยผูกบัญชี LINE ระบบจะเปิดหน้าจอให้กรอกรหัส **MDID (รหัส ว. หรือเบอร์มือถือตัวเลข 4-10 หลัก)** * หากรหัส MDID ตรงกับฐานข้อมูลและยังไม่มีใครผูกบัญชี ระบบจะทำการล็อกรหัส `line_uid` และชื่อ `line_displayname` เข้ากับแถวข้อมูลนั้นทันที
   * หากรหัส MDID นั้นเคยถูกผู้อื่นผูกไปแล้ว ระบบจะแจ้งเตือนความปลอดภัยข้อความสีแดงเพื่อป้องกันสิทธิ์ พร้อมเสนอลิงก์ให้กดแจ้งความประสงค์ขอรีเซ็ตสิทธิ์การผูกบัญชีผ่านอีเมลที่ลงทะเบียนไว้

## 🔧 ส่วนที่ 4: คู่มือการใช้งานสำหรับ Superstaff (ผู้ดูแลระบบสูงสุด)

สิทธิ์การเข้าถึงระดับระบบรากฐาน (Superstaff) ได้รับสิทธิ์สูงสุดในการเข้าควบคุม แก้ไข และกู้คืนข้อมูลดิบในโครงสร้างตารางฐานข้อมูลหลักชีต `staff` ผ่านหน้ากากแอปพลิเคชันโดยตรงเพื่อความปลอดภัยของข้อมูล

### 1. การค้นหาและเข้าถึงประวัติบุคลากรเชิงลึก (Database Fetching)
เมื่อกดเมนูความปลอดภัยสีแดง **"🔧 จัดการข้อมูลบุคลากร"** ระบบจะเปิดแผงควบคุมระบบหลังบ้าน:
* ป้อนรหัส MDID ของบุคลากรที่ต้องการแก้ไขในช่องค้นหาแล้วกดปุ่ม "ดึงข้อมูล"
* ระบบจะสแกนหาแถวข้อมูลจากแผ่นงาน Google Sheets และดึงข้อมูลเชิงลึกทั้งหมดขึ้นมาแสดงผลในฟอร์ม พร้อมแสดงภาพโปรไฟล์ไลน์ปัจจุบัน และระบุเวลาการเข้าใช้งานโปรแกรมล่าสุด (**Last Login Timestamp**) เพื่อใช้ตรวจสอบสถานะการมีตัวตนของพนักงาน

### 2. การควบคุมและอัปเดตฟิลด์ข้อมูลสิทธิ์ผู้ใช้งาน (R/W Field Management)
Superstaff สามารถแก้ไขข้อมูลและเขียนทับตัวแปรระบบได้ทุกช่องอย่างอิสระผ่านหน้าจอ UI โดยมีคำแนะนำในการจัดการฟิลด์สำคัญดังนี้:
* **MDID / Name / Institute:** ข้อมูลระบุตัวตนและสังกัดคณะแพทย์ประจำโรงพยาบาล
* **Role (บทบาทระบบ):** สามารถพิมพ์เปลี่ยนระดับสิทธิ์เพื่อย้ายกลุ่มทำงาน เช่น เปลี่ยนจาก `resident3` เป็น `resident4` หรือปรับสิทธิ์ขึ้นเป็น `instructor` หรือ `staff` เพื่อเปิดฟีเจอร์การทำงานของเครื่องมือใหม่ๆ
* **เป้าหมายหัตถการรายชั้นปี (R2 / R3 / R4):** ช่องกำหนดจำนวนตัวเลขเคสเป้าหมายขั้นต่ำของแต่ละชั้นปีในหลักสูตร ซึ่งส่งผลโดยตรงต่อการคำนวณเปอร์เซ็นต์ความสำเร็จบนหน้าจอ Dashboard ของสตาฟฟ์และหน้า Progress ของผู้เรียน
* **Admission Year / Status:** ระบุปีการศึกษาที่เข้าเรียน และสถานะปัจจุบัน (เช่น ป้อนค่า `ACTIVE` เพื่อเปิดให้ใช้งาน หรือพิมพ์เปลี่ยนค่าเป็น `ALUMNI` เพื่อย้ายสถานะแพทย์ที่สำเร็จการศึกษาแล้วให้สลับเป็นโหมดเปิดอ่านประวัติอย่างเดียว)
* **LINE UID / LINE Display Name (ระบบกู้คืนบัญชีผู้ใช้):** ฟิลด์เก็บรหัสระบบของไลน์ผู้ใช้งาน *เทคนิคสำคัญ:* หากผู้ใช้งานทำโทรศัพท์มือถือหาย, เปลี่ยนเครื่องใหม่, หรือลงทะเบียนผิดพลาดจนระบบแจ้งเตือนว่าผูกบัญชีซ้ำ ให้ Superstaff ทำการ **"ลบข้อความรหัสในช่อง LINE UID นี้ออกจนว่างเปล่า"** แล้วกดบันทึกข้อมูล ระบบจะตัดการเชื่อมโยงบัญชีเดิมทันที (Unbind) ส่งผลให้แพทย์คนดังกล่าวสามารถนำมือถือเครื่องใหม่มากดกรอกรหัสลงทะเบียนผูกบัญชีเริ่มใช้งานใหม่ได้ทันทีอย่างไร้รอยต่อ
* **Advisor MDID (ระบุอาจารย์ที่ปรึกษา):** ป้อนรหัสตัวเลข MDID ของอาจารย์ผู้รับหน้าที่ดูแลแพทย์ประจำบ้านคนนี้ *เทคนิค:* หากแพทย์คนดังกล่าวมีอาจารย์ที่ปรึกษาร่วมหลายท่าน ให้พิมพ์รหัสอาจารย์แพทย์คั่นด้วยเครื่องหมายจุลภาค (Comma) โดยห้ามเว้นวรรค (เช่น `24105,11582,30541`) ระบบจะทำการแยกสิทธิ์และเปิดแผงอนุมัติฟอร์ม 360 องศาให้ไปปรากฏในกล่องงานของอาจารย์แพทย์ทั้ง 3 ท่านพร้อมกันทันที
* **Eval Target / View Target (ขอบเขตการเข้าถึงข้อมูล):** ระบุรหัสตัวเลข MDID ของลูกศิษย์หรือแพทย์ในกลุ่มเป้าหมายคั่นด้วยเครื่องหมายจุลภาค เพื่อกำหนดสิทธิ์เฉพาะให้อาจารย์แพทย์ท่านนั้นๆ สามารถมองเห็น ดึงประวัติ หรือเปิดดูรายงานสถิติตามการแบ่งกลุ่มหน้างานจริง

เมื่อ Superstaff ทำการปรับปรุงข้อมูลเสร็จสิ้นแล้ว ให้กดปุ่ม **"บันทึกข้อมูล"** เพื่อส่ง Payload ผ่านคำสั่ง `update_staff_full` กลับไปทำการอัปเดตบรรทัดข้อมูลบน Google Sheets แผ่นงานจริงให้เรียบร้อย และกดปุ่ม "ค้นหาใหม่" เพื่อล้างหน้าจอเตรียมจัดการรายชื่อถัดไป

## 📊 ภาคผนวก: สรุปตารางโครงสร้างฐานข้อมูลและดัชนีคอลัมน์ (Google Sheets Backend Index)

สำหรับทีมผู้ดูแลระบบคอมพิวเตอร์และสารสนเทศ ตัวแปรฟิลด์ข้อมูลหลังบ้านในแผ่นงานหลักชื่อชีต `staff` มีการจัดวางดัชนีคอลัมน์เริ่มต้นจากลำดับที่ 0 (Index Base 0) ดังต่อไปนี้ เพื่อความถูกต้องในการพัฒนาระบบซอฟต์แวร์ต่อยอด:

| ตัวอักษรคอลัมน์ | ลำดับดัชนี (Index) | ชื่อตัวแปรฟิลด์ระบบ | รายละเอียดประเภทข้อมูลคุณลักษณะ |
| :---: | :---: | :--- | :--- |
| **Column A** | Index 0 | `mdid` | คีย์หลักประจำตัวบุคลากร (Primary Key) ตัวเลข 4-10 หลัก |
| **Column B** | Index 1 | `name` | ชื่อ-นามสกุลจริง ภาษาไทย (เช่น นายแพทย์...) |
| **Column C** | Index 2 | `institute` | ชื่อสถาบันหรือโรงพยาบาลต้นสังกัด (เช่น รามาธิบดี) |
| **Column D** | Index 3 | `role` | สิทธิ์ระบบงาน (resident2, resident3, instructor, staff, ฯลฯ) |
| **Column E** | Index 4 | `r2` | ตัวเลขเป้าหมายหัตถการรวมประจำชั้นปีที่ 2 |
| **Column F** | Index 5 | `r3` | ตัวเลขเป้าหมายหัตถการรวมประจำชั้นปีที่ 3 |
| **Column G** | Index 6 | `r4` | ตัวเลขเป้าหมายหัตถการรวมประจำชั้นปีที่ 4 |
| **Column H** | Index 7 | `line_uid` | รหัสตรวจสอบความปลอดภัยสากลของแอปพลิเคชัน LINE (ขึ้นต้นด้วย U...) |
| **Column I** | Index 8 | `line_displayname` | ชื่อโปรไฟล์หน้าแอปพลิเคชัน LINE ณ ช่วงเวลาผูกสิทธิ์ระบบ |
| **Column J** | Index 9 | `email` | ที่อยู่อีเมลสถาบันสำหรับการจัดส่งลิงก์กู้คืนระบบความปลอดภัย |
| **Column K** | Index 10 | `mobile` | หมายเลขโทรศัพท์เคลื่อนที่สำหรับกรณีติดต่อเร่งด่วน |
| **Column L** | Index 11 | `admission_year` | ปีการศึกษาที่ได้รับการตอบรับเข้าเรียนในสถาบันฝึกอบรม |
| **Column M** | Index 12 | `status` | ตัวบ่งชี้สิทธิ์เปิดปิดระบบงานสถานะบุคคล (`ACTIVE` หรือ `ALUMNI`) |
| **Column N** | Index 13 | `advisor_mdid` | **ฟิลด์เก็บรหัสอาจารย์ที่ปรึกษา (รองรับ String คั่นด้วยเครื่องหมาย `,`)** |
| **Column O** | Index 14 | `eval_target` | ขอบเขตรายชื่อ MDID แพทย์เป้าหมายสำหรับการทำแบบประเมินคะแนน |
| **Column P** | Index 15 | `view_target` | ขอบเขตรายชื่อ MDID แพทย์ประจำบ้านที่อนุญาตให้อาจารย์สืบค้นประวัติ |
