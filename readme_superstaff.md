# คู่มือการใช้งานระบบ skinEPA เวอร์ชั่น 2.30 (สำหรับ Superstaff)

สิทธิ์การเข้าถึงระดับระบบรากฐาน (Superstaff) ได้รับสิทธิ์สูงสุดในการเข้าควบคุม แก้ไข และกู้คืนข้อมูลดิบในโครงสร้างตารางฐานข้อมูลหลักชีต `staff` ผ่านหน้ากากแอปพลิเคชันโดยตรง

---

## 🔧 1. ฟีเจอร์หลักสำหรับผู้ดูแลระบบสูงสุด

### 1.1 การค้นหาและเข้าถึงประวัติบุคลากรเชิงลึก (Database Fetching)
กดเมนู **"🔧 จัดการข้อมูลบุคลากร"**:
* ป้อนรหัส MDID แล้วกด "ดึงข้อมูล" ระบบจะแสดงข้อมูลเชิงลึก ภาพโปรไฟล์ไลน์ และเวลาเข้าใช้งานล่าสุด (**Last Login Timestamp**) เพื่อตรวจสอบสถานะบุคคล

### 1.2 การควบคุมและอัปเดตฟิลด์ข้อมูลสิทธิ์ผู้ใช้งาน (R/W Field Management)
Superstaff สามารถเขียนทับตัวแปรระบบได้ทุกช่องอย่างอิสระ:
* **Role (บทบาทระบบ):** ปรับเปลี่ยนสิทธิ์ เช่น ย้ายจาก `resident3` เป็น `resident4` หรือปรับเป็น `instructor`
* **เป้าหมายหัตถการรายชั้นปี (R2 / R3 / R4):** กำหนดเป้าหมายขั้นต่ำของแต่ละชั้นปี
* **Status:** ป้อน `ACTIVE` เพื่อใช้งาน หรือ `ALUMNI` เพื่อย้ายสถานะผู้จบการศึกษา (โหมดอ่านอย่างเดียว)
* **LINE UID (ระบบกู้คืนบัญชีผู้ใช้):** หากผู้ใช้งานเปลี่ยนเครื่องหรือผูกบัญชีผิด ให้ **"ลบข้อความรหัสในช่อง LINE UID นี้ออกจนว่างเปล่า"** แล้วกดบันทึก ระบบจะตัดการเชื่อมโยง (Unbind) ทันที เปิดทางให้ผู้ใช้เริ่มลงทะเบียนใหม่ได้ด้วย RamaID ไร้รอยต่อ
* **Advisor MDID:** ระบุรหัสอาจารย์ที่ปรึกษา *เทคนิค:* หากมีหลายท่าน ให้พิมพ์รหัสคั่นด้วยคอมมาโดยห้ามเว้นวรรค (เช่น `24105,11582`) ระบบจะกระจายสิทธิ์ให้ทุกท่านอนุมัติฟอร์ม 360 ได้
* **Eval Target / View Target:** ระบุ MDID เป้าหมายคั่นด้วยคอมมา เพื่อจำกัดขอบเขตการเข้าถึงประวัติของอาจารย์

## 📊 2. สรุปตารางโครงสร้างฐานข้อมูล (Google Sheets Backend Index)
ฟิลด์ข้อมูลหลังบ้านในแผ่นงานหลัก `staff` อ้างอิง Index ดังนี้:

| คอลัมน์ | Index | ชื่อตัวแปร | รายละเอียด |
| :---: | :---: | :--- | :--- |
| **A** | 0 | `mdid` | คีย์หลักประจำตัวบุคลากร |
| **B** | 1 | `name` | ชื่อ-นามสกุลจริง |
| **C** | 2 | `institute` | ชื่อสถาบันต้นสังกัด |
| **D** | 3 | `role` | สิทธิ์ระบบงาน |
| **E-G** | 4-6 | `r2`, `r3`, `r4` | เป้าหมายหัตถการรวมประจำชั้นปี |
| **H** | 7 | `line_uid` | รหัสความปลอดภัย LINE |
| **I** | 8 | `line_displayname` | ชื่อโปรไฟล์ LINE |
| **J** | 9 | `email` | ที่อยู่อีเมล |
| **K** | 10 | `mobile` | หมายเลขโทรศัพท์ |
| **L** | 11 | `admission_year` | ปีการศึกษาที่เข้าเรียน |
| **M** | 12 | `status` | สิทธิ์เปิดปิดระบบ (`ACTIVE`, `ALUMNI`) |
| **N** | 13 | `advisor_mdid` | รหัสอาจารย์ที่ปรึกษา (รองรับ String คั่นด้วย `,`) |
| **O** | 14 | `eval_target` | ขอบเขต MDID แพทย์เป้าหมายสำหรับการทำแบบประเมิน |
| **P** | 15 | `view_target` | ขอบเขต MDID สำหรับการสืบค้นประวัติ |
